# analysis.py

`analysis.py` は，追跡済みデータ（ `trackers.npz` ）を読み込み，ミツバチの個体間相互作用（栄養交換）や育児行動を検出し，統計グラフやヒートマップとして可視化するための解析スクリプトである．

<ProjectStructure highlights={["analysis.py", "AssignBeeHive.py"]} />

## 概要

このスクリプトは，個体の向きや部位間の距離，時間的な連続性を考慮して行動を定義する．解析結果は `bees.pkl` に保存され，後のダッシュボード表示や統計比較に利用される．

## 行動検出アルゴリズムの詳細

特定の生物学的行動を検出するために，以下の処理が行われる．

### 1. 栄養交換行動 (Trophallaxis)
`detect_trophallaxis` 関数は，個体間の頭部の近接と相対的な向きに基づき，栄養交換を判定する．

<Steps>
#### 頭部の空間クラスタリング
`DBSCAN` アルゴリズムを用い，全個体の頭部座標をスケーリングした上でクラスタリングする．これにより，物理的に近接している個体ペアを高速に抽出する．

#### 対面判定
近接しているペアに対し， `calc_unit_vector` で算出された各個体の向きベクトルを合成する．ベクトルの和のノルムが閾値 `radian` を下回る（＝互いに向き合っている）場合を候補とする．

#### 時間的持続性の確認
内部関数 `_handle_proximity` と `handle_non_proximity_end` が状態を管理する．近接状態が `start_dur`（デフォルト1秒相当）以上継続した場合に「栄養交換」と認定し， `noise_dur` 以下の短時間の離脱は同一イベントとして扱う．
</Steps>

### 2. 育児行動 (Caring)
`detect_caring` 関数は，個体と巣穴の位置関係から育児を判定する．

*   **位置判定**: 腹部のキーポイントが `AssignBeeHive.pos2id` を通じて特定の巣穴マスク内にあるかを確認する．
*   **姿勢判定**: 頭部と腹部の距離が体長の1/4以下である場合，幼虫に給餌しているような育児姿勢であるとみなす．

## 主要な補助関数

*   **check_overlap_2(individuals, threshold)**: 全個体間でOKS（Object Keypoint Similarity）を計算し，重複して検出されている個体のペアを特定する．
*   **calc_unit_vector(d)**: Numbaで高速化されており，頭，胸，腹のキーポイントのうち，検出されている部位の組み合わせに応じて個体の前進方向ベクトルを算出する．
*   **mark_losted_trackers(frame, trackers, ids_prev, losted)**: 追跡が途切れた個体を「DEAD」としてフレーム上に描画し，解析から除外された個体を可視化する．
*   **gen_hive_heatmap(hive, img_w_sum, path_out)**: 巣穴ごとの利用頻度を `jet` カラーマップで可視化する．内部で `_replace_color_numba` を用いてピクセル単位の高速な色置換を行っている．

## 解析の実行方法

`kpdetect` 関数を呼び出すことで，解析パイプラインが起動する．

<Steps>
#### 環境の準備
`analysis.py` を実行する前に， `tracking.py` によって出力された `trackers.npz` が `output/[動画名]/` に存在することを確認する．

#### スクリプトの実行
```bash
python analysis.py
```

#### 引数の設定
`if __name__ == "__main__":` ブロック内の `kpdetect` を編集する．
*   **第一引数**: 動画ファイル名．
*   **第二引数**: 巣穴定義名．
*   **第三引数**: 使用するYOLOモデル．
*   **draw_trajectory**: `True` にすると，解析動画に移動軌跡が描画される．
</Steps>

## 出力結果

解析完了後，以下のファイルが生成される．

| ファイル名 | 内容 |
| :--- | :--- |
| `bees.pkl` | 全個体の行動履歴，移動距離，相互作用データを含むオブジェクト |
| `hive_heatmap.png` | どの巣穴が頻繁に利用（育児）されたかを示すヒートマップ |
| `trajectories.png` | 個体ごとの総移動距離を比較するグラフ |
| `exchanged_map.png` | 個体間の栄養交換頻度を示すヒートマップ |
| `[filename].mp4` | 行動判定の結果（円やID）をオーバーレイした可視化動画 |

<Callout type="warning">
解析には `AssignBeeHive` の定義が必要なため，あらかじめ `sources/hives/` 内に対応する `.pickle` ファイルが作成されている必要がある．
</Callout>
